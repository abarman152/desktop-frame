# <feature> specification

## Overview

What the feature is and the user value, in a paragraph. Link the parent PRD or epic.

## User stories

The stories this feature delivers, in the form: as a <persona>, I want <capability>, so that <outcome>.

## Behavior

How the feature behaves, including states, edge cases, and what happens on error or when a permission is denied.

## Acceptance criteria

Testable conditions that define done.

## Design and interaction

Link the UI specification. Note the HIG considerations.

## Out of scope

What this feature does not include.

## Dependencies

What must exist first. Other features, services, or permissions.

## Completion checklist
- [ ] User stories tie to personas and jobs.
- [ ] Behavior covers edge cases and error states.
- [ ] Testable acceptance criteria written.
- [ ] UI specification linked.
- [ ] Out of scope and dependencies stated.

## Review checklist
- [ ] Buildable and verifiable as written.
- [ ] No implementation detail that belongs in the technical spec.
- [ ] Meets DocumentationStandards.
